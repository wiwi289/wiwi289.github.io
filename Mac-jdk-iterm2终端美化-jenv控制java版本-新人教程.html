七.使用jenv管理jdk版本

很多时候不同的项目需要不同的jdk版本，否则会抛has been compiled by a more recent version of the Java Runtime (class file version 55.0), this version of the Java Runtime only recognizes class file versions up to 52.0这类似的异常，如果每次都去通过.zshrc来配置jdk版本，未免显得有点太鸡肋了，下面展示通过jenv来管理局部(或者说以项目为单位)jdk版本

  1. 同样的，通过homebrew安装jenv

安装成功后会提示你将下面的命令添加到配置文件，我们这里是~/.zshrc，这步非常重要，否则后续的切换jdk版本不会生效

  1. 查看当前可配置的jdk版本

如上图所示，首次安装只会显示，system选项，行首的*表示当前使用的什么版本

  1. 将前面下载好的jdk添加到jenv管理列表，上面演示的是jdk 12，当前的状态只是下载好了，jenv还不可管理

可以通过open /Library/Java/JavaVirtualMachines，命令进入到jdk安装目录，然后进入到对应的Home目录，拖到终端就可以看到上述的路径

  1. 添加成功后，下面进行jdk版本切换，分global和local只是命令不同，注意local优先级高于global，也就是说如果项目的jdk版本(也就是local)和global版本不同，最终会以local为准
